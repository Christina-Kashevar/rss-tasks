TypeScript


by Christina Kashevar for RS School 2020

TypeScript has been increasing in its popularity for the last couple of years. It’s mentioned among the five most promising languages in 2020. Angular, one of the largest frontend frameworks, is using TypeScript. About 60% of JS programmers already use TypeScript, and 22% wish to try.

The development of TypeScript began at the end of 2012. Although it originated in Microsoft, and its actual creator is programmer Anders Hejlsberg, but this project immediately began to develop as OpenSource.

Why TypeScript?

  • a strongly typed language => reduces the potential for errors
  • implements many of the concepts that are inherent in object-oriented languages
  • faster and easier development of complex programs
  • an open source project. Github
  • provides planned features from future JavaScript editions

Types

TypeScript has a variety of basic types, like:

  • Boolean
  • Number
  • String
  • Array
  • Tuple
  • Enum
  • Any
  • Unknown
  • Null
  • Undefined
  • Void
  • Never
  • Object

The colon character is used to set the type. Examples of creating variables:


              let x: number = 10; 
              let hello: string = "hello world";
              let isValid: boolean = true;
              let a: undefined = undefined;
              let b: null = null;
              let person: object = {name:"Tom", age:23};

Arrays are defined using the [ ] expression and are also strongly typed. That is, if the array initially contains strings, then in the future it will only be able to work with strings.


            let colors: string[] = ["red", "green", "blue"];

Tuple types allow us to express an array with a fixed number of elements whose types are known, but need not be the same. 


            let userInfo: [string, number];
            userInfo = ["Tom", 28];

            userInfo = [28, "Tom"]; // ERROR

Enum is intended to describe a set of numeric data using string constants. 


            enum Season { Winter, Spring, Summer, Autumn };
            let current: Season = Season.Summer;
            console.log(current); // 2

Any is specified data, the type of which may not be known at the time of writing the application.


            let someVar: any = "hello";
            someVar = 20;

We may need to describe the type of variables that we do not know when we are writing an application. In these cases, we use the unknown type.


              let notSure: unknown = 4;
              notSure = "maybe a string instead";
              // OK, definitely a boolean
              notSure = false;

Unknown is a lot like any, but it won’t let us perform operations on the variable before it is explicitly type-checked


  let looselyTyped: any = 4;
  looselyTyped.toFixed();

  let strictlyTyped: unknown = 4;
  strictlyTyped.toFixed(); // ERROR: Object is of type 'unknown'

Void is the absence of having any type at all. We may commonly see this as the return type of functions that do not return a value:


            function warnUser(): void {
              console.log("This is my warning message");
            }

The never type represents the type of values that never occur.


// Function returning never must not have a reachable end point
function error(message: string): never {
  throw new Error(message);
}

// Inferred return type is never
function fail() {
  return error("Something failed");
}

// Function returning never must not have a reachable end point
function infiniteLoop(): never {
  while (true) {}
}

TypeScript allows to define type aliases using the type keyword:


            type stringOrNumberType = number | string;

Functions

When we define a function we can specify the types of each one of the parameters as well as the type of the returned value.


              function add(a: number, b: number): number {
                return a + b;
              }

When defining a function we can specify default values for any of its parameters.


function getName(firstName: string, lastName: string = "Geller ") {
  return firstName + " " + lastName;
}

If it is necessary for the function to accept a set of parameters of the same type, then the spread operator is used, followed by an array: 


function addNumbers(firstNumber: number, ...numberArray: number[]): number {
  let result = firstNumber;
  for (let i = 0; i < numberArray.length; i++) {
      result+= numberArray[i];
  }
  return result;
}

classes

TypeScript takes an object-oriented approach and has full class support. The class represents a template for creating objects and encapsulates the functionality that an object must have.


            class User {
              id: number;
              name: string;
              getInfo(): string {
                  return "id:" + this.id + " name:" + this.name;
              }
            }

After defining the class, we can create its objects:


              let alice: User = new User();
              alice.id = 2;
              alice.name = "Alice";

In addition to ordinary functions, classes have special functions - constructors, which are defined using the constructor keyword.


class User {

  id: number;
  name: string;
  constructor(userId: number, userName: string) {

      this.id = userId;
      this.name = userName;
  }
  getInfo(): string {
      return "id:" + this.id + " name:" + this.name;
  }
}
let alice: User = new User(2, "Alice");

Access modifiers allow you to hide the state of an object from external access and control access to that state. There are three modifiers in TypeScript: public, protected, and private.

If no modifier is applied to properties and functions of classes, then such properties and functions are treated as if they were defined with the public modifier.

If the private modifier is applied to properties and methods, then they cannot be accessed from the outside when creating an object of this class.

The protected modifier is in many ways similar to private - properties and methods with this modifier are not visible from the outside, but they can be accessed from inherited classes

Interface

An interface defines the properties and methods that an object must implement. In other words, an interface is a definition of a custom data type, but without implementation. 


              interface IUser {
                id: number;
                name: string;
              }
            
              let employee: IUser = {
                id: 1, 
                name: "Tom"
               }

When defining an interface, we can set some properties as optional using a question mark. It is not necessary to implement such properties:


              interface IUser {
                id: number;
                name: string;
                age?: number;
              }

              let employee: IUser = {
                id: 1, 
                name: "Alice",
                age: 23
              }

              let manager: IUser = {
                id: 2, 
                name: "Tom"
              }

Also, an interface can contain read-only properties whose value cannot be changed. Such properties are defined using the readonly keyword:


              interface Point {
                readonly x: number;
                readonly y: number;
              }

              let p: Point = { x: 10, y: 20 };
              p.x = 5; // error

Interfaces can be implemented not only by objects, but also by classes. The implements keyword is used for this:


interface IUser {
  id: number;
  name: string;
  getFullName(surname: string): string;
}

class User implements IUser{

  id: number;
  name: string;
  age: number;
  constructor(userId: number, userName: string, userAge: number) {

      this.id = userId;
      this.name = userName;
      this.age = userAge;
  }
  getFullName(surname: string): string {

      return this.name + " " + surname;
  }
}

let tom = new User(1, "Tom", 23);
console.log(tom.getFullName("Simpson"));
